# モデル確認と調整ガイド

## ✅ 現在の状態

- ✅ SnakeModelがServerStorageに配置済み
- ⏳ モデルの構造確認と調整が必要

## 🔍 ステップ3: モデルの構造確認と調整

### 1. モデルの構造を確認

1. **ExplorerパネルでSnakeModelを展開**
   - SnakeModelをクリックして展開
   - モデル内の構造を確認

2. **確認すべき項目**:
   - Humanoidが含まれているか
   - パーツの数と構造
   - アニメーションが含まれているか

### 2. Humanoidの削除（必要に応じて）

**Humanoidが含まれている場合**:

1. SnakeModel内の「Humanoid」を探す
2. Humanoidを選択
3. Deleteキーを押すか、右クリック > Delete

**理由**: プレイヤーの既存のHumanoidを使用するため、モデル内のHumanoidは不要です。

### 3. アニメーションの削除（オプション）

**アニメーションが含まれている場合**:

1. SnakeModel内の「Animations」フォルダやアニメーションオブジェクトを探す
2. 必要に応じて削除（MVPでは削除を推奨）

### 4. モデルの位置とサイズの確認

1. SnakeModelを選択
2. Propertiesパネルで以下を確認：
   - Position（位置）
   - Size（サイズ）
   - 必要に応じて調整

**注意**: モデルは後でコードでサイズ調整されるため、基本的なサイズで問題ありません。

## 🧪 ステップ4: 動作確認

### 1. ファイルの同期確認

1. **Rojoサーバーが起動していることを確認**
   ```bash
   # ターミナルで確認
   lsof -i :34872
   ```

2. **Explorerパネルで以下の構造が表示されていることを確認**:
   ```
   ServerScriptService
     └── Server
         ├── Services
         │   └── PlayerManager.luau
         └── Components
             └── VenomousCharacter.luau
   ```

### 2. ゲームを実行

1. **「Play」ボタンをクリック**
2. **Outputウィンドウを開く**（View > Output）
3. **エラーメッセージを確認**

### 3. 期待される動作

- ✅ エラーが表示されない
- ✅ プレイヤーが毒ヘビに変身する
- ✅ プレイヤーのサイズが初期サイズ（1.0）に設定される

### 4. 問題が発生した場合

**モデルが表示されない**:
- Outputウィンドウで「Model not found in ServerStorage: SnakeModel」という警告がないか確認
- SnakeModelの名前が正確に「SnakeModel」になっているか確認

**エラーが発生する**:
- Outputウィンドウのエラーメッセージを確認
- モデルの構造が想定と異なる可能性があります

## 📝 次のステップ

動作確認が完了したら：

1. ✅ モデルが正しく表示される
2. ✅ サイズが正しく設定される
3. ⏳ 移動システムの実装に進む
