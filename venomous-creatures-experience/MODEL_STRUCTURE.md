# SnakeModelの構造について

## 📋 確認された構造

```
SnakeModel
  └── Mesh
```

## 🔍 構造の詳細

この構造は、SnakeModelがModelオブジェクトで、その中にMeshコンポーネントが含まれていることを示しています。

**注意**: Meshは通常、PartやMeshPartの子コンポーネントです。実際の構造を確認するには：

1. **SnakeModelをさらに展開**
   - ExplorerパネルでSnakeModelをクリック
   - 中にPartやMeshPartがあるか確認

2. **想定される実際の構造**:
   ```
   SnakeModel (Model)
     └── Part または MeshPart
         └── Mesh
   ```

## ✅ 実装の対応

現在の実装は以下のように対応しています：

1. **モデルのロード**: ServerStorageからSnakeModelを取得
2. **パーツの検出**: モデル内のすべてのBasePart（Part、MeshPartなど）を検出
3. **位置の調整**: HumanoidRootPartの位置に合わせて配置
4. **サイズの調整**: すべてのパーツのサイズを調整

## 🧪 動作確認

### 1. モデルの構造を完全に確認

1. **ExplorerパネルでSnakeModelを完全に展開**
   - すべての子要素を表示
   - PartやMeshPartがあるか確認

2. **Propertiesパネルで確認**
   - 各要素のタイプを確認
   - サイズや位置を確認

### 2. ゲームを実行

1. **「Play」ボタンをクリック**
2. **Outputウィンドウを確認**
   - エラーがないことを確認
   - 警告メッセージを確認

### 3. 期待される動作

- ✅ モデルが正しくロードされる
- ✅ プレイヤーの位置にモデルが配置される
- ✅ サイズが正しく設定される（初期サイズ: 1.0）

## 🐛 問題が発生した場合

### モデルが表示されない

**原因**: モデル内にPartやMeshPartがない可能性

**解決方法**:
1. SnakeModelの構造を完全に確認
2. PartやMeshPartが存在するか確認
3. 存在しない場合は、別のモデルを試すか、Partを追加

### 位置が正しくない

**原因**: モデルの位置調整が正しく動作していない

**解決方法**:
1. Outputウィンドウでエラーメッセージを確認
2. モデルの構造を確認
3. 必要に応じて、位置調整のコードを修正

### サイズが正しく設定されない

**原因**: サイズ調整の処理が正しく動作していない

**解決方法**:
1. Outputウィンドウでエラーメッセージを確認
2. モデル内のパーツが正しく検出されているか確認
3. 開発者コンソール（F9）でサイズを確認

## 📝 次のステップ

モデルの構造を完全に確認したら：

1. ✅ モデルが正しく表示される
2. ✅ サイズが正しく設定される
3. ⏳ 移動システムの実装に進む
