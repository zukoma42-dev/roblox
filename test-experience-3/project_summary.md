# 変身システム開発レポート

## 1. 実装機能まとめ
今回構築した変身システムは、単なるモデルの差し替えにとどまらず、実用性と拡張性を考慮した以下の機能を備えています。

### コアシステム
- **サーバーサイド管理**: チート対策を考慮し、変身ロジックは全てサーバー側で制御。
- **設定駆動型アーキテクチャ**: `TransformationConstants` ファイル一つで新しい変身を追加・設定可能。
- **クールダウン・効果時間管理**: ゲームバランス調整が容易なパラメータ管理。

### 人型変身 (Character Mode)
- **アニメーション同期**: プレイヤーの動きに合わせて変身後のモデルも手足を動かす（R6/R15相互対応）。
- **装備・見た目の継承**: 変身モデルに含まれる服やBodyColorsを適用し、解除時に元の服を復元。
- **物理挙動の最適化**: `Massless` や `CanCollide` の自動設定により、移動速度低下や物理干渉を防止。

### 物体変身 (Object Mode)
- **自動接地 (Auto-Ground)**: モデルのサイズとプレイヤーの身長から、自動的に地面に接地する位置を計算。
- **物理干渉の排除**: プレイヤーが物体に引っかからないよう、当たり判定やRaycast判定を自動無効化。
- **自動Weld**: モデル内のパーツがバラバラにならないよう自動結合。
- **位置・回転調整**: 必要に応じて手動でオフセットや回転を微調整可能。

### UIシステム
- **動的生成**: 設定ファイルに追加された変身を自動的に読み込み、ボタンを生成・整列。

## 2. 開発工数・費用見積もり
このシステムをRoblox Studio専門のエンジニア（コーダー）に依頼した場合の一般的な見積もりです。

### 想定工数 (Man-Hours)
| 工程 | 内容 | 時間 (目安) |
| :--- | :--- | :--- |
| **要件定義・設計** | データ構造、通信仕様、拡張性の設計 | 3時間 |
| **コア実装** | 変身・解除ロジック、透明化処理 | 6時間 |
| **人型・アニメーション対応** | R15/R6マッピング、Weld処理、服の着せ替え | 5時間 |
| **物体・物理演算対応** | AutoGround計算、物理プロパティ自動設定 | 4時間 |
| **UI実装** | 動的リスト生成、イベント連携 | 2時間 |
| **デバッグ・調整** | 浮き・埋まり・物理干渉の修正、テスト | 4時間 |
| **ドキュメント作成** | セットアップガイド、仕様書 | 2時間 |
| **合計** | | **約 26時間** |

### 費用感 (Cost Estimate)
エンジニアのスキルレベルにより単価は異なりますが、実用レベルの品質（バグが少なく、拡張性があるコード）を担保する場合の目安です。

- **時間単価**: $30 〜 $60 (約 4,500円 〜 9,000円)
- **総額目安**: **$800 〜 $1,500 (約 12万円 〜 23万円)**

※ クラウドソーシング等で安価に依頼する場合（$10-$20/h）もありますが、その場合は「拡張性がない（ハードコードが多い）」「バグが多い（物理演算の配慮不足など）」リスクが高まります。今回のコードは「設定ファイルで簡単に追加できる」という拡張性を重視しているため、付加価値が高い設計となっています。
